# [브리프] U-1a: 온보딩 3/3 과목 선택 UI 개선 (사용자 피드백 반영)

> 대상 모델: 오퍼스 권장 · 발행: 페이블 2026-07-05 · 대상: `khu-weaver_v2/` 온보딩 3/3 화면(직접 선택 모드)만

## 사용자 피드백 (스크린샷 기준)
1. 과목 체크 칸이 작고, 목록 영역이 **스크롤이 안 됨** (2학년 중간에서 잘림) — 버그
2. 학년 아코디언 대신 **1·2·3·4학년 4열 그리드**로 한눈에

## 구현
1. 모달을 와이드 레이아웃으로: max-width 1100~1200px(뷰포트 92% 상한), 내부 과목 영역 `overflow-y:auto`로 **반드시 스크롤 가능**하게(모달 높이 = 뷰포트 88% 상한, footer 버튼은 sticky로 항상 노출).
2. 과목 영역: `grid-template-columns: repeat(4, 1fr)` — 열 헤더 "1학년(7과목) · 2학년(11과목) …". 좁은 화면(<900px)은 2열, (<600px)은 1열 폴백.
3. 체크 행 크게: 행 전체가 클릭 영역(label), min-height 44px, hover 배경, 체크 시 버건디 배경+✓(대시보드 노드와 동일 문법). 과목명 word-break: keep-all.
4. 우상단 "n과목 선택 · m학점" 실시간 카운터(학점 합계 추가).
5. 하단 버튼 로직: 0개 선택 시 "선택 없이 로드맵 시작", 1개 이상이면 "n과목 반영하고 시작"으로 라벨 전환.
6. 접근성: 체크박스 키보드 포커스 이동 가능, 열 헤더는 스크롤 시 sticky.

## 제약
- 수정 허용: onboarding.js, index.html(해당 모달), style.css. app.js 계산 로직·localStorage 계약·데이터 파일 금지.
- 검증: node --check + 참조 무결성 대조 + STATUS.md 1줄(| U-1a | ... |) + 사용자 육안확인 요청.

## U-1b (추가 요구, 2026-07-05): 학년별 "전체 선택" 버튼

> U-1a 완료 후 사용자 아이디어. 같은 모달, 같은 제약.

1. 각 학년 열 헤더 우측에 작은 토글 버튼: 기본 라벨 **"전체 선택"**.
2. 동작(3상태):
   - 해당 학년 과목이 하나도/일부만 체크됨 → 클릭 시 그 학년 **전부 체크**
   - 이미 전부 체크됨 → 라벨이 "전체 해제"로 바뀌어 있고, 클릭 시 그 학년 전부 해제
3. 개별 체크 변경 시 버튼 라벨 즉시 동기화. 상단 "n과목 · m학점" 카운터도 함께 갱신.
4. 스타일: 텍스트 버튼(고스트) — 열 헤더의 시각 위계를 해치지 않게. word-break: keep-all.
5. 검증에 시나리오 추가: 전체선택→개별 1개 해제→라벨 "전체 선택" 복귀 확인. STATUS 보고는 U-1b로 1줄.
