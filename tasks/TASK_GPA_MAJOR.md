# [브리프] U-3: 성적 기능을 '전공 평점'으로 축소 (사용자 확정)

> 대상 모델: 오퍼스 권장 · 발행: 페이블 2026-07-05 · 대상: `khu-weaver_v2/`
> 결정 배경: 전체 평점은 교양 입력이 필요해 로드맵 정체성·저마찰 원칙과 충돌 → 전공 과목만으로 계산되는 **전공 평점**으로 범위 확정. 전체 평점은 인포21 영역.

## 구현
1. **성적 모델 변경: 성적은 학기가 아니라 과목에 귀속.** 이수 체크된 전공 과목에 성적(A+/A0/…/F, 4.3 만점) 선택 UI를 부여 — 과목 노드 클릭 상세나 이수 목록에서 드롭다운. 전공 평점 = Σ(성적점수×학점)/Σ학점, 성적 미입력 과목은 계산 제외(P/NP 과목도 제외).
2. 기존 "성적 계산기"(학기 탭 모달)는 **제거 또는 전공 평점 요약 뷰로 대체** — 학기 버킷이 사라지므로 계절학기 학년구분 버그(구 이슈)는 자연 소멸. 학기별 추이 차트가 있다면 함께 제거(전공평점은 학기 무관).
3. 사이드바 KPI 라벨을 "전공 평점 (4.3)"으로. 교양 관련 입력·표시가 어딘가 있으면 전부 제거.
4. localStorage 마이그레이션: 기존 학기 기반 성적 데이터가 저장돼 있으면 과목 귀속 구조로 1회 변환(과목 매칭 실패분은 폐기, 콘솔 경고만).
