# [브리프] U-1: 온보딩 개편 — 저마찰 3단계 (JS 수정 허용)

> 대상 모델: 오퍼스 권장 · 발행: 페이블 2026-07-05
> 작업 폴더: `khu-weaver_v2/` 만. 설계 기준: `khu-weaver_v2/docs/UX_FLOW.md` (반드시 먼저 읽기)
> 이 태스크는 온보딩만. 스크린샷 OCR(A안)은 별도 태스크(TASK_OCR_GEMINI) — 여기서는 **진입 카드와 자리(placeholder)만** 만들고 클릭 시 "준비 중" 대신 B안으로 폴백.

## 범위
- 수정 허용: `js/onboarding.js`, `index.html`(온보딩 영역), `css/style.css`(온보딩 스타일)
- 수정 금지: `js/app.js`의 계산 로직, `js/registry.js`, `js/data/*` (프로필 저장 포맷도 기존 localStorage 키·구조 유지 — 대시보드가 읽는 계약)

## 구현 사항 (UX_FLOW.md의 온보딩 플로우 그대로)
1. 랜딩: 로고+카피 1줄+[시작하기]. 기존 온보딩 첫 화면을 이 구성으로 단순화.
2. 단계 인디케이터(1/3~3/3) + 뒤로가기 + 단계별 입력 localStorage 보존.
3. 1/3 학과: registry의 DEPARTMENTS로 렌더. available:false 학과는 비활성 카드+"준비 중" 배지.
4. 2/3 학번 칩 + 이름 입력(선택 입력, placeholder "이름 (선택)").
5. 3/3 카드 3개(A 스크린샷/B 직접 선택/C 아직 없어요). C 클릭 즉시 대시보드. B는 학년별 과목표 체크 UI — **본인 학년 이하 구간을 기본으로 펼침**(학번으로 현재 학년 추정: 2026-입학년도+1).
6. 완료 후 코치마크 3개(1회 노출, localStorage 플래그).
7. 재방문 시 온보딩 스킵(기존 동작 유지·확인).

## 검증 (필수)
1. `node --check` 수정한 전 JS + 로드 시뮬레이션(CURRICULA 7학번)
2. JS 참조 id/class ↔ HTML/CSS 전수 대조 (기존 QA 스크립트 방식)
3. 신규 유저 시나리오 3종 수동 점검 기록: ①신입생(C 스킵) ②재학생(B 선택) ③재방문 스킵
4. `tasks/STATUS.md` 맨 아래 1줄 보고 (| U-1 | ... |) 후 사용자 육안확인 요청
