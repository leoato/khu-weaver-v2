## F-1. 잔손질 4건 (모델: 소넷)

1-1. **기이수 입력 행 전체 클릭**: 과목 행의 체크박스만이 아니라 행(카드) 어디를 눌러도 선택/해제 토글. label로 감싸거나 행 click 핸들러(단, 성적 드롭다운 클릭은 전파 차단 — 드롭다운 조작이 체크 토글로 이어지면 안 됨).
1-2. **노드 카드 글자 겹침 제거**: 컴팩트 노드에서 "3학점"과 트랙 배지(CSY 등)+✓/🔒 아이콘이 겹침, "수강가능" 배지도 겹침(예: 안테나공학). 절대배치를 flex 배치로 바꾸고 gap 확보, 넘치면 과목명만 말줄임(…) — **상태 배지·학점·트랙 배지는 절대 겹치거나 잘리면 안 됨**.
1-3. **로드맵 가운데 정렬**: "나의 전공 로드맵" 아래 학년 4열 그리드가 좌측으로 치우침 → 컨테이너 max-width + margin:auto로 중앙 정렬(사이드바 있는 그리드 안에서의 중앙 기준).
1-4. **설계 배경 문구 축소·이동**: "KHU-Weaver 설계 배경 및 큐레이션 기작" 섹션을 페이지 최하단으로 이동, 본문은 **1줄 요약**("OP.GG의 데이터 큐레이션 철학을 학사 설계에 이식한 지능형 전공 로드맵 플랫폼" 수준)으로 교체, 글자색 --text-muted. 기존 긴 문단·게임패드 아이콘 박스 제거.

## F-2. OCR 매칭 강화 (모델: 소넷) — 수정 파일: js/ocr.js 만

2-1. **프롬프트에 과목 사전 주입**: buildPrompt()에 현재 프로필 학번의 CURRICULA 과목 전체(이름+학수번호, 51개 내외)를 목록으로 포함하고 지시 추가: "표의 과목이 이 목록의 과목과 사실상 같다면(오타·띄어쓰기·유사표기 포함) 목록의 정식 name과 code로 응답하라. 목록에 없는 과목(교양 등)은 원문 그대로 쓰고 code를 생략하라." → '전자자기학1'→EE201, '파이썬/파이선' 문제를 모델 단에서 해결.
2-2. **클라이언트 매칭도 완화**: 정규화(공백·특수문자·대소문자 제거)에 더해 ①파이썬↔파이선 등 흔한 표기 흔들림을 잡는 자모 단위 유사도(간단 편집거리 ≤1 허용) ②숫자 접미사 보존('전자기학1'≠'전자기학2' — 숫자는 유사도 허용 대상 제외).
2-3. **리사이즈 상향**: 긴 변 1536→2048px (오독 감소, 8장×2048 요청 크기 확인해 20MB 초과 시 품질 0.8로 조정).
2-4. 검증: tests/ocr-mock.html의 목업 응답에 '전자자기학1'·'웹/파이썬프로그래밍' 케이스를 추가해 EE201·SWCON104로 매칭됨을 확인, STATUS에 기록.

## F-3. 탭 개편 + 기이수 입력 접근성 (모델: **오퍼스**) — 구조 변경 큼

> 선행: F-1·F-2 완료 + `js/data/TRENDS.js` 존재(별도 R-1 태스크 산출물). TRENDS.js가 아직 없으면 "출처 수집 중" 빈 상태 UI로 만들고 STATUS에 명시.

3-1. **새 탭 "진로 트렌드"**: 산업계 수요 트렌드를 트랙 탭줄에 새 탭으로 분리. 콘텐츠 = `js/data/TRENDS.js`의 데이터로 **세로 막대그래프**(순수 CSS/SVG, 라이브러리 금지) + 각 막대에 수치 + 하단에 **출처 표기**(기관명·연도·링크) + 5줄 내외 분석 텍스트(TRENDS.js의 analysis 배열 그대로 렌더 — 임의 창작 금지). 기존 대시보드의 트렌드 섹션은 제거.
3-2. **새 탭 "내 분석"**: 졸업 이수율·전공 평점·역량 레이더 카드를 이 탭으로 이동(메인 화면은 로드맵에 집중). 사이드바가 비면 사이드바 자체를 제거하고 로드맵을 전폭으로 — 중앙정렬(1-3)과 함께 자연 해결됨. 헤더의 성적계산기 진입점은 유지.
3-3. **"기이수 교과목 입력" 버튼**: 로드맵(학년 그리드) 좌상단에 버튼 배치 → 클릭 시 온보딩 3/3과 동일한 입력 창(A 스크린샷/B 직접 선택)이 모달로 뜸. **단 C("아직 수강한 과목이 없어요") 카드는 이 진입경로에선 제외.** 기존 체크 상태를 불러와 표시하고, 저장 시 대시보드 즉시 갱신. 페이지 하단의 기존 "기이수 교과목 입력" 섹션은 제거(중복).
3-4. 검증 추가: 탭 전환 3종(전체/진로 트렌드/내 분석) 렌더 확인 시나리오, 모달 열기→체크 변경→반영 시나리오를 STATUS에 기록.
