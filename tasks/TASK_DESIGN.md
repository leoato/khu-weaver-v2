# [브리프] 웹사이트 리디자인 — roadmap.sh형 (D-시리즈)

> 대상 모델: **오퍼스 권장**(디자인 품질), 소넷 가능 · 발행: 페이블 QA 세션 2026-07-04
> 작업 폴더: `03_디자인 시안/` **에만** 산출. `khu-weaver_v1/`(정본)은 읽기만 하고 절대 수정 금지.

## 배경 (이것만 읽으면 됨 — 다른 문서 탐색 금지)
- KHU-Weaver = 전자공학과 이수체계 시각화 웹앱(vanilla JS). 현 디자인은 가독성 문제: 배경 그라데이션 + 이수율·레이더·타임라인이 위계 없이 나열됨.
- 확정된 방향: **roadmap.sh형** — 선후수 노드 트리가 화면의 주인공, 요약 지표는 우측 사이드바로.
- 기준 시안: `03_디자인 시안/dashboard_roadmap_v1.html` (페이블 작성). **이 시안의 디자인 언어를 계승·발전**시킬 것:
  - 노드 상태 3종: 이수완료(버건디 채움+✓) / 수강가능(파랑 테두리) / 잠김(회색 점선+🔒) — 색맹 대응 위해 색+형태 이중 구분
  - 트랙 컬러 6종은 노드 좌측 4px 스트라이프로만 (면적 최소화)
  - 배경: 순수 화이트 계열 + 얇은 그리드, 그라데이션 금지
  - 경희 버건디 hsl(343,66%,33%)가 유일한 브랜드 액센트

## D-1. 나머지 화면 시안 제작 (지금 실행 가능)
같은 디자인 언어로 정적 HTML 시안 3장을 `03_디자인 시안/`에 추가:
1. `onboarding_v1.html` — 학과·학번 선택 온보딩. 정본 `js/onboarding.js`·registry의 흐름(학과 선택→학번 선택) 참고
2. `timeline_v1.html` — 선후수 타임라인 전체 화면(대시보드보다 노드·간선 밀도 높은 버전, SVG 간선 연결 포함)
3. `gpa_modal_v1.html` — 성적 계산기 모달(4.3 만점, 학기 탭). **알려진 UX 버그 반드시 해결**: ① 계절학기가 몇 학년의 여름/겨울인지 구분 안 됨 ② 4-2학기 탭이 화면에 가려짐(가로 스크롤/줄바꿈 처리)
- 각 파일 상단 주석에 디자인 의도 3줄 기록. 더미 데이터 사용(실데이터 로직 구현 금지 — 시안임).
- 완료 후 STATUS.md에 1줄: `| D-1 | 시안 3종 | onboarding/timeline/gpa | 완료 | 특이사항 | 날짜 |`

## D-2. 정본 적용 (게이트: 사용자가 시안 승인한 후에만)
- `khu-weaver_v1/`을 `khu-weaver_v2/`로 복사 후 그 안에서만 CSS/마크업 교체.
- JS 로직(데이터·계산)은 수정 금지, 클래스명 변경 시 app.js 참조 지점 함께 갱신.
- 검증: node --check 전 JS 파일 + 로드 시뮬레이션(window.CURRICULA 7개 학번 주입 확인) + 사용자 육안 확인 요청.
