# 세션 실행 가이드 — 무엇을, 어떤 순서로, 어떤 프롬프트로

> 모든 세션은 Cowork에서 새로 열고 **같은 폴더(`전정대 공모전`)를 연결**한다.
> 크롤링·수정·통합은 **소넷**으로 충분 (페이블 아껴두기).

## 실행 순서 한눈에

```
[지금 가능 — 5개 병렬 OK]                [전부 PASS 후]      [마지막]
 세션1  EE 2024 크롤링 ┐
 세션2  EE 2022 크롤링 ├─ 서로 독립,      세션6 통합(INT)  →  세션7 QA
 세션3  EE 2021 크롤링 │  동시 진행 가능
 세션4  EE 2020 크롤링 │
 세션5  EE_2023 버그수정 ┘
```

---

## 세션 1~4 — 학번별 데이터 구축 (모델: 소넷)

- **세션 제목**: `KHU-Weaver EE 2024 데이터` (연도만 바꿔서 4개)
- **프롬프트** (이거 한 줄 그대로, 연도만 교체):
  > tasks/TASK_CRAWL_EE.md를 읽고 **2024**학번 담당으로 그대로 수행해. 브리프가 지정한 파일 외에는 읽지 마.
- **목표**: `js/data/EE_2024.js` 생성 + `validate_curriculum.js` PASS + `tasks/STATUS.md`에 1줄 보고
- **완료 판단**: 세션이 "PASS + STATUS 기록 완료"라고 하면 끝. FAIL을 못 넘기면 특이사항을 STATUS에 남기게 하고 종료 → QA 세션에서 처리
- **다음 단계**: 남은 크롤링 세션 실행. 4개 다 끝났으면 세션 6으로

## 세션 5 — EE_2023 버그 수정 (모델: 소넷)

- **세션 제목**: `KHU-Weaver 2023 버그수정`
- **프롬프트**:
  > tasks/TASK_FIX_2023.md를 읽고 그대로 수행해. 브리프가 허용한 부분 외에는 수정 금지.
- **목표**: EE324 참조 오류 해결(과목 추가 또는 선수항목 삭제, 2023 PDF 근거) + 검증 PASS + STATUS 보고
- **다음 단계**: 세션 1~4와 무관하게 아무 때나 실행 가능. 끝나면 세션 6 조건 하나 충족

## 세션 6 — 통합·등록 v1.6 (모델: 소넷)

- **선행조건**: STATUS.md에 C-2020/2021/2022/2024 + F-2023 전부 PASS
- **세션 제목**: `KHU-Weaver v1.6 통합`
- **프롬프트**:
  > tasks/STATUS.md에서 선행조건 확인 후 tasks/TASK_INTEGRATE.md를 그대로 수행해. 조건 미달이면 중단하고 보고해.
- **목표**: registry.js 연도 추가, index.html script 태그 추가, PATCHNOTES v1.6, 전체 검증 PASS
- **다음 단계**: 세션 7 (QA)

## 세션 7 — 최종 QA (모델: 페이블 = 이 세션 이어서)

- **방법**: 새 세션 말고 **기존 페이블 세션(이 세션)에 이어서** 요청:
  > 파이프라인 전부 끝났어. STATUS.md 확인하고 QA 검수해줘.
- **목표**: 전 파일 검증, registry/로드순서 시뮬레이션, 데이터 샘플 대조, 이슈 있으면 수정 브리프 발행
- **다음 단계**: v1.6 확정 → 베타테스트 설계(새 계획, 페이블)
