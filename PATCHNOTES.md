## v2.0 (2026-07-07) — 03_디자인 시안 전체 적용 + 실제 상호작용 구현 (KHU-Weaver v2)

D-3(마크업 재배치)까지는 정적 배치만 시안에 맞췄으나, 이번 버전은 03_디자인 시안 4종(로드맵/타임라인/GPA모달/온보딩)의 **실제 동작**까지 JS로 구현했다. 사용자 요청으로 이번 버전부터 v2 폴더의 JS 수정 금지 규칙을 해제.

### 추가 (Added)
- **선수과목 기반 노드 상태 자동 판정** (`js/app.js`): `getDirectPrereqs()`/`getCourseState()` 신규 — 직접 선수과목이 전부 이수되었으면 `avail`(파랑 테두리+"수강가능"), 하나라도 미충족이면 `locked`(회색 점선+🔒). 기존 D-3에서 CSS만 준비해두고 비활성 상태였던 `.course-avail`/`.course-locked` 훅이 이번에 실제로 부여됨.
- **노드 클릭 고정(핀) 하이라이트**: 기존 호버 시 선후수 체인 하이라이트는 유지하고, 클릭하면 마우스가 빠져나가도 하이라이트가 고정되도록 `applyHighlight`/`clearHighlight`/`pinnedNode` 로직 추가. 같은 노드 재클릭 또는 빈 공간 클릭 시 해제.
- **GPA 모달 계절학기 학년별 탭**: `SEMESTERS`를 학년별 1·2학기(정규)+여름·겨울(계절) 16탭 구조로 전면 교체(기존 '기타' 단일버킷 제거 — D-2 STATUS에 남아있던 "계절학기 학년구분 버그①" 해결). 탭 바를 가로 스크롤+좌우 페이드+화살표 버튼으로 재구성(시안 gpa_modal_v1 계승), 학년 그룹 라벨 삽입.
- **온보딩 2단계 위저드**: 기존 드롭다운 3종 단일 폼 → STEP1(학과 카드 선택, 준비중 학과는 잠금 노드로 표기)·STEP2(학번 칩 그리드+이름+실시간 미리보기) 2단계로 전면 재작성(`js/onboarding.js`). DEPARTMENTS/ADMISSION_YEARS는 시안처럼 하드코딩하지 않고 registry.js에서 계속 동적으로 읽음.
- **헤더 액션 버튼 2종**: "성적 계산기"(기존 GPA 모달 오픈 재사용) · "내 빌드 공유"(이수 현황을 텍스트로 요약해 클립보드 복사, 백엔드 없는 간단 구현).

### 변경 (Changed)
- **타임라인 레이아웃 전면 개편** (`index.html`, `js/app.js`, `css/style.css`): 학년당 고정 6슬롯 그리드(트랙별 행 고정 배치)를 폐지하고, 트랙 우선순위→과목코드 순 정렬 후 학년 레인에 순서대로 쌓는 자유배치로 전환. 카드도 한 줄 컴팩트 스타일(높이 ~32px)로 재설계해 최대 20과목/학년(전 학번 공통 최댓값)이 스크롤 없이 한 화면에 들어오도록 함.
- `css/style.css`: `.timeline-course-card`를 flex 한 줄 레이아웃으로 재작성, 상태 3종 배지(`::after`)를 카드 높이에 무관하게 수직 중앙 정렬로 변경, `.year-courses-list` gap 축소.

### 검증 (Verified)
- `node --check`: registry.js·데이터 7파일·onboarding.js 정상. app.js는 이번 세션의 bash 샌드박스 마운트 지연 이슈로 원본 경로에서 직접 검증이 안 돼, 동일 내용을 임시 경로에 복사해 `node --check` 통과 확인(내용은 Read 도구로 전문 대조 완료).
- registry.js·js/data/는 khu-weaver_v1과 diff 결과 무변경 확인(교육과정 데이터 자체는 이번 작업과 무관).
- index.html의 모든 `getElementById` 참조(app.js+onboarding.js)가 실제 존재하는 id와 1:1 매칭됨을 grep 대조로 확인.
- CURRICULA 7개 학번(2020~2026) 로드 시뮬레이션 정상, 폴백 정상.

### 알려진 제약 (Known Limitations)
- "내 빌드 공유"는 서버/링크 발급 없이 클립보드 복사 방식의 간단 구현(요청 시 실제 공유 URL 발급 등으로 확장 가능).
- avail/locked 판정은 "직접 선수과목"만 확인(간접 연쇄까지 재귀 확인하는 recursive 판정은 아님) — 데이터 구조상 prerequisites가 직접 관계 위주로 기록되어 있어 실용적으로 충분하다고 판단.

## v1.2 (설계안) — 대규모 학과/학번 데이터 확장(Scalability) 아키텍처 수립

향후 다양한 학번(21~26학번)과 타 학과(컴퓨터공학과, 생체의공학과 등) 확장에 대비하여 데이터베이스 구조와 작업 파이프라인을 확립했습니다.

### 데이터 아키텍처 개편안 (진행 예정)
- **파일 분할 (Modularization):** 모든 교육과정 데이터를 하나의 파일에 넣지 않고, `js/data/` 폴더 내에 `학과코드_학번.js` (예: `EE_2023.js`) 형태로 분리 보관합니다.
- **네임스페이스 연동:** 각 분리된 데이터 파일은 전역 변수 `window.CURRICULA["학과코드"]["학번"]`에 자신의 데이터를(과목, 선수조건, 졸업학점 등) 주입(Inject)합니다.
- **index.html 로드 순서:** `<script src="js/data/EE_2023.js">` 등을 먼저 로드한 후, 메인 로직(`app.js`)을 로드하여 온보딩에서 선택된 정보를 바탕으로 데이터를 즉시 렌더링합니다.

### 학과 및 학번 추가 작업 파이프라인 (Data Pipeline)
웹 크롤링의 기술적 한계(로그인 월, 비표준 PDF 포맷)를 극복하기 위해 AI 지원 반자동 방식을 채택합니다.
1. **수집:** 팀원들이 타겟 학번/학과의 **'표준이수형태(학년별 개설표)'**와 **'선수과목 일람표'** 이미지를 캡쳐합니다.
2. **AI 파싱:** Gemini 등 AI 모델에 이미지를 업로드하고, KHU-Weaver 표준 JSON 배열 포맷(`courses`, `prerequisites`)으로 출력을 지시합니다.

## v1.4 핫픽스 (2026-05-30) — 구조 정합성 버그 수정

여러 버전에 걸쳐 누적된 파일 구조 결함 3종을 정리했습니다. 기능 동작·디자인 변경 없이 결함만 수정.

### 수정 (Fixed)

- **레지스트리 잔재 정리** (`js/data/EE_2025.js`): 이 데이터 파일 안에 `window.DEPARTMENTS` / `ADMISSION_YEARS` / `getCurriculum` 가 중복 정의되어 `registry.js`(정본)와 충돌 소지가 있던 부분 제거. 이제 EE_2025.js는 `EE_2023.js`와 동일하게 자기 데이터만 `window.CURRICULA["EE"]["2025"]`에 주입. 학과/학번 목록과 조회 함수는 `registry.js`가 단독 소유.
- **파일 끝 NUL 바이트 제거** (`js/data/EE_2025.js`): 파일 말미에 채워져 있던 널 문자 1,058개 제거(구문 오류 유발).
