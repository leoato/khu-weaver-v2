# KHU-Weaver UX 플로우 설계 v1

> 작성: 페이블 2026-07-05 · 구현 브리프: `tasks/TASK_UX_ONBOARD.md`(온보딩), `tasks/TASK_OCR_GEMINI.md`(스크린샷 입력)

## 설계 원칙 (전 화면 공통)

1. **화면당 결정 1개.** 한 화면에서 사용자가 고를 것은 딱 하나. 설명 문장은 화면당 최대 2줄.
2. **3분 안에 첫 가치.** 온보딩 완료(→내 로드맵 확인)까지 클릭 수 목표: 신입생 4클릭, 재학생(스크린샷) 6클릭 내외.
3. **언제든 건너뛰기.** 어떤 단계도 막다른 길이 되면 안 됨 — 스킵하면 나중에 대시보드에서 채울 수 있다는 안내 1줄.
4. **진행 상태 상시 표시.** 상단에 `1/3 → 2/3 → 3/3` 점 인디케이터. 뒤로가기 항상 가능, 입력값은 단계 이동해도 보존(localStorage).
5. **선택지는 카드로, 텍스트는 단어 단위 줄바꿈**(word-break: keep-all — 적용 완료).

## 온보딩 플로우 (첫 접속)

```
[0. 랜딩]  로고 + 한 줄 카피 + [시작하기]
   ↓
[1/3 학과]  학과 카드 선택 (EE 활성, BME/CSE "준비 중" 비활성 카드로 노출 — 확장 예고 효과)
   ↓
[2/3 학번·이름]  학번(입학년도) 칩 선택 + 이름 입력 1칸 (이름은 표시용, 선택 입력)
   ↓
[3/3 이수 이력]  "지금까지 들은 과목을 알려주세요" — 카드 3개 중 택1
   ├─ A. 📷 에타 스크린샷으로 한 번에   ← 추천 배지. 최대 8장, 성적까지 자동 입력
   ├─ B. ✅ 직접 선택하기              → 학년별 과목표에서 체크 (2학년이면 1~2학년 구간만 먼저 펼침)
   └─ C. 🌱 아직 수강한 과목이 없어요   → 신입생용 스킵. 바로 대시보드
   ↓
[완료]  대시보드(학년별 과목표=로드맵) 진입
        + 첫 진입 1회만 코치마크 3개(노드 상태 의미 / 사이드바 이수율 / 성적계산기 위치). "다시 보지 않기" 기본 체크.
```

### A안(스크린샷) 세부 — TASK_OCR_GEMINI
```
사진 첨부(1~8장, 드래그앤드롭+버튼)
 → [분석 시작] → 직조(Weaving) 애니메이션 + 단계 문구 순환 + n/8 진행
 → 결과 확인표: 인식된 과목·성적 목록, 매칭 실패 항목은 노랑 표시로 수동 지정
 → [적용] → 대시보드 (성적은 GPA 계산기에도 자동 반영)
```
- 개인정보 안내 1줄: "사진은 분석에만 쓰이고 저장되지 않아요. 이름·학번은 가리고 찍어도 됩니다."
- 실패/오류 시: "직접 선택하기(B)"로 자연스럽게 폴백 — 막다른 길 금지 원칙.

## 재방문 플로우
localStorage에 프로필 있으면 온보딩 전체 스킵 → 바로 대시보드. 헤더 칩(학과·학번) 클릭 시에만 재설정 진입.

## 확정 결정 (2026-07-05)
- **성적 = 전공 평점만.** 교양 입력 없음(전체 평점은 인포21 영역). 성적은 학기가 아닌 과목에 귀속 → 학기 탭 계산기 제거, 계절학기 구분 문제 자연 소멸. 상세: `tasks/TASK_GPA_MAJOR.md`
- 3/3 직접 선택 화면은 학년 아코디언 → **4열 그리드**(와이드 모달, 스크롤 필수). 상세: `tasks/TASK_UX_ONBOARD_FIX.md`

## 하지 않기로 한 것 (산만함 방지)
- 온보딩 중 트랙(진로) 질문 — 대시보드에서 자연스럽게 탐색하도록. 질문이 늘면 피로.
- 튜토리얼 팝업 연속 노출 — 코치마크 3개 1회로 제한.
- 추천 시간표 — 에타 영역(기 확정 방침).
