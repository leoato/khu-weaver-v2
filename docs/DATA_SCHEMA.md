# KHU-Weaver 교육과정 데이터 표준 스키마 v1.0

> 작성: 2026-07-04 (페이블 세션) · ROADMAP 3-1 산출물
> **모든 신규 데이터 파일은 이 문서를 유일한 기준으로 작성한다.**
> 작성 후 반드시 `node tools/validate_curriculum.js <파일>` 통과 필수.

---

## 1. 파일 규칙

- 경로/이름: `js/data/<학과코드>_<학번>.js` (예: `EE_2024.js`, `BME_2026.js`)
- 인코딩: UTF-8, 들여쓰기 4칸, 기존 파일(`EE_2026.js`)과 동일한 코딩 스타일
- 파일 상단 주석에 **출처 명시 필수**: PDF명, 등록일, 어느 별표에서 어떤 필드를 얻었는지
  (EE_2026.js 상단 주석 형식을 그대로 따를 것)

## 2. 주입 구조 (보일러플레이트)

```js
window.CURRICULA = window.CURRICULA || {};
window.CURRICULA["EE"] = window.CURRICULA["EE"] || {};

window.CURRICULA["EE"]["<학번>"] = {
    gradReq: 130,
    courses: [ ... ],
    prerequisites: [ ... ],
    tracks: { ... }
};
```

## 3. 필드 정의

### 3.1 `gradReq` (number, 필수)
최저 졸업이수학점. 출처: 교육과정 시행세칙 제11조(또는 해당 조항).

### 3.2 `courses[]` (필수, 1개 이상)
출처: [별표1] 교육과정 편성표 (+ 1·2학기 동시개설 과목은 [별표3] 주개설학기 기준 1개 슬롯 배치)

| 필드 | 타입 | 허용값 | 설명 |
|---|---|---|---|
| `code` | string | `/^[A-Z]{2,6}[0-9]{3,4}$/` | 학수번호 (예: EE213, AMTH1009, SWCON104) |
| `name` | string | 비어있지 않음 | 과목명 (PDF 표기 그대로, 공백 없이) |
| `type` | string | `"전공 기초"` `"전공 필수"` `"전공 선택"` | 이수구분 — **띄어쓰기 포함 정확히 이 3개만** |
| `credits` | number | 0~4 정수 | 학점 (졸업논문 0학점 허용) |
| `year` | number | 1~4 | 개설 학년 |
| `sem` | number | 1 또는 2 | 개설 학기 |
| `track` | string | 4.1의 track 코드 | 레이더 역량 축 분류 |
| `isMinor` | boolean | 생략 또는 `true` | 보조 과목 표시 (타임라인 축소 표시용). 해당 시에만 기입 |

### 3.3 `prerequisites[]` (필수, 없으면 빈 배열)
출처: [별표4] 선수과목 지정표

```js
{target:"EE371", prereqs:["AMTH1004", "EE211"]},  // 대상과목 ← 선수과목(복수면 모두 수강)
```

- `target`: 대상 과목 code — 반드시 courses에 존재
- `prereqs`: 선수 과목 code 배열 — 각각 courses에 존재, target 자신 포함 금지
- target당 항목 1개 (선수 여러 개면 prereqs 배열에 나열)
- 각 항목 뒤에 한글 주석으로 `// 과목명 ← 선수과목명` 병기 (EE_2026.js 스타일)

### 3.4 `tracks{}` (필수)
출처: [별표2] 전공심화 분류 + 앱의 진로 트랙 5종 매핑

```js
tracks: {
    semiconductor: [...],  // 반도체/공정
    embedded: [...],       // 임베디드/IoT
    ai: [...],             // 인공지능/신호처리
    telecom: [...],        // 통신/신호처리
    circuit: [...]         // 회로/시스템 설계
}
```

- **EE는 위 5개 키 고정** (임의 추가/삭제 금지 — app.js가 이 키를 참조)
- 배열 원소는 모두 courses에 존재하는 code
- 타 학과 확장 시: 해당 학과의 트랙 키 체계를 **먼저 이 문서에 추가 정의**한 뒤 데이터 작성 (app.js/레이더 축 수정 동반 — 임의 진행 금지)

## 4. 코드표

### 4.1 track 코드 (EE)

| 코드 | 의미 |
|---|---|
| `MATH` | 수학 |
| `PHYS` | 물리 |
| `COMP` | 프로그래밍/컴퓨터 |
| `CS` | 통신·신호(Communication/Signal) |
| `CSY` | 회로·시스템(Circuit/System) |
| `SW` | 반도체·파동(Semiconductor/Wave) |

※ 주의: `SW`는 소프트웨어가 아니라 **반도체·파동**임. `CS`도 컴퓨터과학 아님.

### 4.2 학과 코드
학교 공식 분류를 따름 (ROADMAP 3-2): `EE` 전자공학과 · `BME` 생체의공학과 · `CSE` 컴퓨터공학과

## 5. 신규 데이터 추가 체크리스트

1. [ ] PDF 원문에서 별표1/별표3/별표4/별표2 + 졸업이수학점 조항 확보
2. [ ] 이 스키마대로 `js/data/<학과>_<학번>.js` 작성 (기존 파일 수정 금지, 새 파일만)
3. [ ] `node tools/validate_curriculum.js js/data/<파일>.js` → PASS 확인
4. [ ] 검증 출력의 이수구분별 학점 합계가 PDF의 편성표 합계와 일치하는지 눈으로 대조
5. [ ] `tasks/STATUS.md`에 결과 기록 (파일명, 과목 수, 선수 건수, 검증 결과, 특이사항)
6. [ ] **registry.js / index.html은 수정하지 않는다** — 통합 세션(TASK_INTEGRATE)이 일괄 처리

## 6. 자주 하는 실수 (검증기가 잡아주는 것들)

- type 오타: `"전공기초"`(붙여씀) ❌ → `"전공 기초"` ⭕
- 선수과목이 해당 학번 편성표에 없는 과목을 참조 (학번별로 과목 구성이 다름!)
- track 코드에 임의 값 사용
- 1·2학기 동시개설 과목을 2개 슬롯으로 중복 등록 → code 중복 에러
