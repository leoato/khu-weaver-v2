# 중간계획서 제출 전 체크리스트

## 배포

- GitHub 저장소에 최신 파일을 올렸다.
- Netlify에서 GitHub 저장소를 연결했다.
- Build command는 비워두고, Publish directory는 `.`로 설정했다.
- Netlify Environment variables에 `GEMINI_API_KEY`를 등록했다.
- 배포된 URL에서 첫 화면이 정상 표시되는지 확인했다.
- 온보딩에서 학과/학번 선택과 이수 과목 반영이 되는지 확인했다.
- 에타 또는 인포21 성적 스크린샷 업로드 분석이 되는지 확인했다.
- `POST_DEPLOY_SMOKE_TEST.md`의 항목을 모두 확인했다.

## API 키 보안

- `js/config.js`의 `GEMINI_API_KEY`는 빈 문자열이다.
- 실제 키는 Netlify 환경변수에만 있다.
- `.env` 파일은 GitHub에 올리지 않았다.
- 예전에 `config.js`에 실제 키를 넣어 공유한 적이 있다면, 기존 키를 폐기하고 새 키를 발급했다.

## 계획서에 넣을 내용

- 체험 URL을 `MIDTERM_PLAN_WEB_SECTION.md`의 `체험 URL` 자리에 입력한다.
- API 키 보호 방식은 “Netlify Function 서버리스 프록시 + 환경변수”라고 적는다.
- 업로드 이미지는 저장하지 않고 분석 요청에만 사용한다고 적는다.
- 심사위원이 확인할 핵심 기능은 로드맵, 이수 과목 반영, 전공 평점/졸업 요건 분석, OCR 자동 입력으로 정리한다.

## 심사위원 안내 문구

웹 링크 접속 후 학과와 입학연도를 선택하면 개인화된 전공 이수체계 로드맵을 확인할 수 있습니다. 성적 스크린샷 업로드 기능을 사용하면 OCR 분석을 통해 이수 과목과 성적이 자동 반영되며, API 키는 서버리스 프록시와 환경변수로 보호되어 클라이언트에 노출되지 않습니다.
